# Evolution Report — Trading Mind Care

## 产品设计全景

### 核心循环

```
计划 → 交易 → 复盘 → 弱点提取 → 预警 → 计划（闭环）
```

### 架构

```
┌─────────────────────────────────────────┐
│  Browser / pywebview                     │
│  (Vanilla HTML + CSS + JS)              │
└──────────────┬──────────────────────────┘
               │ HTTP (localhost:18088)
┌──────────────▼──────────────────────────┐
│  FastAPI (Uvicorn)                       │
│  ├── /api/plans        计划 CRUD         │
│  ├── /api/reviews      复盘 + AI 拷打    │
│  ├── /api/vulnerabilities 弱点矩阵       │
│  ├── /api/settings     配置管理          │
│  └── /api/notifications 飞书通知         │
├──────────────────────────────────────────┤
│  SQLite (WAL mode)                       │
│  ~/.trading_mind_care/mind_care.db       │
├──────────────────────────────────────────┤
│  APScheduler                             │
│  ├── 每日 00:05 弱点权重衰减             │
│  └── 每日 08:30 飞书推送                 │
├──────────────────────────────────────────┤
│  LLM Adapter (OpenAI-compatible API)     │
│  ├── 复盘拷打（交易教练 persona）         │
│  └── 弱点提取（JSON 结构化输出）          │
└──────────────────────────────────────────┘
```

### 打包方案

- **开发阶段**：`python run.py` → 自动打开浏览器
- **发布阶段**：PyInstaller onefile → `TradingMindCare.exe`
  - 静态文件通过 `datas` 打入 exe
  - 运行时通过 `sys._MEIPASS` 定位资源
  - 数据库存储在用户目录，升级不丢数据

### 关键设计决策

详见 `DECISIONS.md`，核心调整：
1. 去掉复盘表 UNIQUE 约束（支持多 session）
2. 弱点衰减改为每日一次（避免频繁复盘导致过度衰减）
3. pywebview 延后到打包阶段
4. 飞书推送增加启动时补推逻辑

## 迭代中的取舍

1. **未实现流式输出**：当前 LLM 调用为非流式。流式对 UX 有提升但增加前端复杂度，留作 v2。
2. **未实现文件锁单实例保护**：跨平台文件锁实现差异大，暂用端口占用自然防重复。
3. **未实现计划编辑**：当前只有添加和删除。编辑功能优先级低于核心闭环。
