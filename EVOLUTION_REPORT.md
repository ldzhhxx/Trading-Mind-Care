# Evolution Report — Trading Mind Care

## 产品设计全景

### 核心循环

```
计划 → 交易 → 复盘 → 弱点提取 → 预警 → 计划（闭环）
```

### 架构

```
┌─────────────────────────────────────────┐
```

### 打包方案

- **开发阶段**：`python run.py` → 自动打开浏览器
- **发布阶段**：PyInstaller onefile → `TradingMindCare.exe`
  - 静态文件通过 `datas` 打入 exe
  - 运行时通过 `sys._MEIPASS` 定位资源
  - 数据库存储在用户目录，升级不丢数据

## v4.0.0 商用级打磨 (2026-06-01)

### 稳定性改进
- **SQLite 并发安全**：添加 `PRAGMA busy_timeout=5000` 和连接超时 30s，防止并发写入时报错
- **LLM 流式错误处理**：`call_llm_stream` 现在正确捕获 HTTP 错误、超时、网络断开，返回友好中文错误信息
- **LLM 配置缓存**：60s TTL 缓存避免每次调用都查数据库
- **月报 f-string 修复**：修复 `monthly.py` 中会导致运行时崩溃的语法错误
- **衰减率安全范围**：decay_rate 限制在 0.8-1.0，防止用户误配导致数据异常

### 代码质量
- **结构化日志**：全局 logging 配置，关键操作有日志记录
- **异常日志**：未处理异常自动记录完整 traceback
- **移除死代码**：删除 `data.py` 中未使用的 `shutil` 导入
- **依赖版本锁定**：`requirements.txt` 使用精确版本号
- **Docstring 补全**：所有公共函数添加文档字符串

### 打包修复
- **APScheduler hiddenimports**：PyInstaller spec 添加 `apscheduler` 相关模块，修复打包后定时任务不工作的问题

### 版本号
- 升级至 v4.0.0，反映商用级打磨完成

## 设计决策

详见 `DECISIONS.md`，核心调整：
1. 去掉复盘表 UNIQUE 约束（支持多 session）
2. 弱点衰减改为每日一次（避免频繁复盘导致过度衰减）
3. pywebview 延后到打包阶段
4. 飞书推送增加启动时补推逻辑
5. LLM 调用失败时优雅降级（保存复盘但标记 AI 不可用）
6. SQLite busy_timeout 替代应用层锁（更简单可靠）
